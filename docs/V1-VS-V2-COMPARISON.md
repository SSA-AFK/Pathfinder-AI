## 二、量化对比

### 2.1 情绪曲线平滑度

| 指标 | V1（单次调用） | V2（链式 4 步） | 优劣 |
|---|---|---|---|
| 情绪序列 | `[7, 6, 5, 4, 8, 6, 8]` | `[7, 5, 6, 5, 8, 6, 8]` | — |
| 相邻差值和（越小越平滑） | 1+1+1+4+2+2 = **11** | 2+1+1+3+2+2 = **11** | 持平 |
| **最大单次跳变** | **4**（等待生成 4→查看结果 8） | **3**（等待生成 5→查看结果 8） | **V2 胜** |
| 是否满足 PRD 连续性要求（≤3） | ❌ 不满足 | ✅ 满足 | **V2 胜** |
| 运行时校验拦截 | 无（V1 不做连续性校验） | 有（`_check_emotion_continuity`，跳变>3 触发重试） | **V2 胜** |

### 2.2 阶段划分

| 指标 | V1 | V2 |
|---|---|---|
| 阶段数 | 7 | 7 |
| 阶段列表 | 发现产品、了解功能、首次试用、等待生成、查看结果、迭代优化、导出复用 | 发现产品、注册账号、首次使用、等待生成、查看结果、迭代优化、导出复用 |
| 是否覆盖生命周期 | 是（发现→了解→使用→导出） | 是（发现→注册→使用→导出） |
| 阶段粒度差异 | 用通用的"了解功能"跳过注册环节 | 将"注册账号"独立为一个阶段，捕获关键流失点 |

### 2.3 中间产物可追溯性

| 指标 | V1 | V2 |
|---|---|---|
| 中间产物 | 无（一次性输出） | 4 步 traces，可追踪每步输出 |
| 调试友好度 | ❌ 黑盒，输出异常时无法定位哪一步出错 | ✅ 出问题时精确到 Step N |
| 面试展示价值 | 低 | **高**（可展示分步推理过程） |

## 三、定性对比

### 3.1 痛点具体化程度

V1 描述风格偏泛化，缺少量化细节：
- "首页信息密度高，用户难以快速判断产品是否匹配自身需求"
- "功能描述偏技术化，缺少场景化表达导致理解门槛高"
- "导出 Mermaid 代码在 Confluence 或 Notion 等平台可能渲染异常"

V2 的描述更具体，包含可测量的细节：
- "难以在 **5 秒内** 判断产品是否匹配自身需求"（量化时间）
- "注册流程有 **3 个步骤**，每多一步就多一层流失风险"（量化步骤）
- "导出的 Mermaid 代码在 **部分知识库平台** 渲染异常"（更精确）

### 3.2 摘要质量

**V1 摘要**（160 字）：
> "用户整体旅程呈现典型的SaaS产品探索曲线：从好奇探索（7分）到经历等待焦虑的谷底（4分），最后在获得价值后情绪回升（8分）。"

- 仅描述曲线形状，无洞见
- 未给出任何改进方向

**V2 摘要**（184 字）：
> "用户旅程呈现典型的探索-挫折-回升曲线：从发现产品的好奇（7分）经历注册摩擦（5分）和等待焦虑（4分谷底），在获得有价值结果后情绪强劲反弹（8分）。**关键体验断点在注册环节的摩擦和等待阶段的不确定性，改善这两处将显著提升整体旅程满意度。**"

- 描述曲线 + 指明关键断点
- 给出可落地的改进方向
- 总结中包含具体阶段的分数参照

## 四、架构性差异分析

### 4.1 V1：单次调用（One-shot）

```
用户输入 → Prompt 模板 → LLM 一次调用 → JSON 输出
```

**特征**：
-  简单，一次 API 调用即完成
- ❌ AI 需要同时满足 5 种格式要求和 4 种内容约束，容易顾此失彼
- ❌ 缺少中间验证，输出异常时无法定位根因
- ❌ 无运行时情绪连续性校验

### 4.2 V2：链式分步（Chain-of-thought）

```
用户输入
  → Step 1: 阶段拆解（专注框架选择）
  → Step 2: 行为+痛点（专注用户视角描述）
  → Step 3: 情绪评分+机会点（专注评分合理性）
  → Step 4: 汇总+Mermaid（前 3 步结果整合）
  → 最终 JSON
```

**特征**：
-  分而治之，每步只有一个聚焦任务
- ✅ 每步产出可验证（字段校验 + 连续性校验）
- ✅ 验证失败自动重试（最多 3 次/步）
- ✅ 中间产物全程可追溯（traces 返回给前端）
-  4 次 API 调用，耗时更长（但 mock 下无差异）

### 4.3 根本原因

**V2 的链式推理强制模型先结构化思考再填充细节，避免了 V1 的幻觉和情绪跳跃。**

V1 一次性要求 AI 同时完成"划分阶段 + 填充行为 + 评分情绪 + 生成图表"，多任务竞争使 AI 难以在每个维度都保持高质量。V2 将任务分解为 4 个独立步骤，每步聚焦单一目标：

- Step 1 只思考"用什么框架划分阶段"，不会被行为描述分散注意力
- Step 2 站在用户视角写行为和痛点，不需要考虑评分
- Step 3 看到完整的阶段行为后才能做出合理的情绪评分（信息差消除）
- Step 4 整合所有中间产物，只需做组装而非从头推理

这就是为什么 V2 的痛点更具体、摘要更有洞见、情绪评分更连续 —— 不是因为用了不同的模型，而是因为用更好的流程引导了模型的推理路径。
